# GeomWrapper

GeomWrapper — это настольное приложение и алгоритмический прототип для построения триангулированной оболочки вокруг входной STL-геометрии на основе метода декартового фронта.

Проект реализует полный вычислительный конвейер:
- загрузка и предварительная обработка STL-модели
- построение знакового поля расстояний на декартовой сетке
- выделение фронта в окрестности исходной поверхности
- построение начальной триангулированной оболочки
- итерационное стягивание оболочки к исходной геометрии
- поддержка равномерного и адаптивного октодеревного режимов

Приложение предоставляет графический интерфейс для настройки параметров, визуализации промежуточных этапов и сохранения итоговой оболочки в формате STL.

## Назначение проекта

GeomWrapper предназначен для построения замкнутой триангулированной оболочки вокруг произвольной полигональной поверхности, заданной STL-моделью.

Основная идея метода состоит в том, чтобы не строить сетку напрямую по исходной поверхности, а использовать более устойчивую и управляемую схему:
1. представить исходную геометрию на декартовой фоновой сетке
2. выделить фронт в узкой окрестности поверхности
3. построить начальную оболочку по фронту
4. итерационно деформировать оболочку в направлении исходной геометрии

Такой подход позволяет получить структурированную и контролируемую оболочку, пригодную для дальнейшей обработки, анализа и визуализации.

## Общая схема алгоритма

### 1. Загрузка STL

На вход подаётся полигональная геометрия в формате STL.  
Модель загружается и преобразуется во внутреннее представление поверхности, используемое на всех последующих этапах.

### 2. Построение знакового поля расстояний

Вокруг исходной модели строится декартова сетка с настраиваемыми параметрами:
- шаг сетки
- размер отступа от модели
- ограничение на максимальное разрешение

На этой сетке вычисляется знаковое поле расстояний. По нему определяется, какие области относятся к геометрии, внешнему пространству и фронту.

### 3. Выделение фронта

В окрестности нулевого уровня знакового расстояния выделяется узкая полоса.

На основе этой полосы определяется набор фронтовых ячеек. Именно они задают дискретное декартово приближение оболочки вокруг исходной поверхности.

### 4. Построение начальной оболочки

Поддерживаются два режима построения оболочки.

#### Равномерный режим
Начальная оболочка строится непосредственно по фронтовым ячейкам равномерной декартовой сетки.

#### Адаптивный октодеревный режим
Фоновое пространство адаптивно разбивается октодеревом. После балансировки по правилу 2:1 выделяются фронтовые листья, выполняется согласование фронта, после чего строится оболочка по открытым patch-областям адаптивных ячеек.

### 5. Итерационное стягивание оболочки

После построения начальной оболочки выполняется итерационная деформация, приближающая оболочку к исходной геометрии.

Этап стягивания сочетает:
- регуляризацию формы оболочки
- притяжение к исходной поверхности
- ограничение смещений по локальному масштабу сетки

В результате стартовая декартова оболочка преобразуется в поверхность, более точно повторяющую форму исходной модели.

### 6. Визуализация и сохранение результата

Приложение позволяет визуализировать:
- исходную STL-модель
- фронтовые кубы
- начальную оболочку
- итоговую оболочку после стягивания

Финальная оболочка может быть сохранена в формате STL.

## Назначение основных модулей

### `main.py`

Точка входа в приложение. Запускает графический интерфейс.

### `app/gui.py`

Графический интерфейс приложения.

**Отвечает за:**
- выбор STL-файла
- настройку параметров алгоритма
- запуск вычислительного конвейера
- визуализацию исходной модели, фронта и оболочек
- сохранение итоговой оболочки

### `app/config.py`

Содержит описание структуры параметров приложения.

**Включает:**
- параметры декартовой сетки
- параметры октодерева
- параметры этапа стягивания
- объединённый набор параметров для вычислительного конвейера

### `app/mesh_io.py`

Модуль загрузки входной геометрии.

**Назначение:**
- чтение STL-файла
- преобразование модели во внутреннее представление поверхности

### `app/sdf_grid.py`

Модуль построения декартовой фоновой сетки и вычисления знакового поля расстояний.

**Отвечает за:**
- построение сетки вокруг модели
- учёт шага сетки, отступа и ограничения по разрешению
- вычисление знакового расстояния в ячейках сетки

### `app/front.py`

Модуль выделения фронта для равномерного режима.

**Отвечает за:**
- построение масок по знаковому полю расстояний
- выделение внешней области
- выделение фронтовых ячеек
- построение визуализируемого набора front-cubes

### `app/shell.py`

Модуль построения начальной оболочки в равномерном режиме.

**Назначение:**
- формирование триангулированной оболочки по фронтовым ячейкам регулярной декартовой сетки

### `app/octree.py`

Модуль адаптивного разбиения пространства.

**Отвечает за:**
- построение листьев октодерева
- адаптивное уточнение в окрестности фронта
- 2:1 балансировку дерева
- выделение фронтовых листьев
- построение геометрии front-cubes для визуализации

### `app/front_repair.py`

Модуль согласования и локального исправления дефектов фронта в адаптивном режиме.

**Назначение:**
- анализ открытых patch-границ фронта
- обнаружение дефектных конфигураций
- локальное уточнение окрестности проблемных участков
- подготовка согласованного фронта перед построением оболочки

### `app/shell_octree.py`

Модуль построения оболочки в адаптивном октодеревном режиме.

**Отвечает за:**
- обработку открытых patch-областей фронтовых листьев
- выделение контуров patch
- локальную триангуляцию patch-областей
- построение итоговой начальной оболочки для octree-режима

### `app/shrink.py`

Модуль итерационного стягивания оболочки к исходной геометрии.

**Назначение:**
- деформация стартовой оболочки
- регуляризация сетки в процессе деформации
- притяжение оболочки к целевой поверхности
- ограничение смещений по локальному масштабу

### `app/pipeline.py`

Основной вычислительный конвейер проекта.

**Объединяет этапы:**
- загрузка STL
- построение знакового поля расстояний
- выделение фронта
- построение начальной оболочки
- итерационное стягивание
- возврат результатов вычисления

### `app/metrics.py`

Модуль расчёта метрик качества сетки.

**Позволяет вычислять:**
- число вершин
- число треугольников
- длины рёбер
- площади треугольников
- угловые характеристики
- расстояния до целевой поверхности

## Параметры

### Параметры сетки

- **pitch** — шаг декартовой сетки
- **padding mul** — коэффициент внешнего отступа от модели
- **band mul** — множитель ширины полосы фронта
- **max dim** — ограничение на максимальное разрешение сетки

### Параметры октодерева

- **octree max level** — максимальный уровень адаптивного разбиения
- **balance iters** — число итераций балансировки октодерева

### Параметры стягивания

- **iters** — число итераций стягивания
- **step** — базовый шаг деформации
- **constraint mul** — ограничение максимального смещения вершины
- **lap iters per step** — число итераций дополнительной релаксации
- **lap relax** — коэффициент релаксации

## Результаты работы

В ходе выполнения алгоритма строятся следующие объекты:

- **исходная поверхность** — входная STL-модель
- **фронт** — декартовое приближение области вокруг поверхности
- **начальная оболочка** — оболочка, построенная по фронту
- **итоговая оболочка** — результат итерационного стягивания

Для оболочек могут вычисляться метрики качества:

- количество вершин
- количество треугольников
- минимальная, средняя и максимальная длина ребра
- минимальная, средняя и максимальная площадь треугольника
- минимальный, средний и максимальный угол
- среднее и максимальное расстояние до целевой поверхности

## Пример рабочего сценария

1. Пользователь загружает STL-модель.
2. Программа строит декартову фоновую сетку вокруг объекта.
3. На сетке вычисляется знаковое поле расстояний.
4. По узкой полосе вокруг поверхности выделяется фронт.
5. По фронту строится начальная триангулированная оболочка.
6. Оболочка итерационно стягивается к исходной поверхности.
7. Итоговый результат отображается в окне визуализации и может быть сохранён.
